# LeanSMT Native Binding Sources

This directory contains the LeanSMT native wrapper and JNI bridge sources used by JavaSMT.

## Files

- `leansmt_wrapper.h` and `leansmt_wrapper.c`:
  C wrapper over Lean/solver-native APIs.
- `leansmt.i`:
  SWIG interface source of truth for generated JNI bindings.
- `leansmt_wrap.c`:
  SWIG-generated JNI C wrapper.
- `leansmt_extra_jni.c`:
  JNI helpers for operations that need custom conversions.
- `build-runtime.sh`:
  Linux x64 helper that stages a reproducible local LeanSMT runtime for JavaSMT.
- `lean-smt.lock`:
  Pinned upstream LeanSMT revision used by the local runtime build.

## Building the local runtime

The supported local build path is:

```bash
./lib/native/source/libleansmt/build-runtime.sh /absolute/path/to/lean-smt
```

This script:

- copies the upstream checkout into `build/lean-smt-work` instead of mutating it
- injects the JavaSMT `SmtJNI.lean` overlay from `build/build-publish-solvers/leansmt-overlay/`
- stages the real runtime files in `build/leansmt-staging/x64`
- refreshes LeanSMT symlinks in `lib/native/x86_64-linux`

The build is intentionally limited to Linux x64. On macOS hosts, run it inside a Linux VM or
container.

## Regenerating SWIG bindings

Run from anywhere:

```bash
./lib/native/source/libleansmt/regenerate-swig-bindings.sh
```

The script regenerates:

- `lib/native/source/libleansmt/leansmt_wrap.c`
- `src/org/sosy_lab/java_smt/solvers/leansmt/LeanSMT.java`
- `src/org/sosy_lab/java_smt/solvers/leansmt/LeanSMTJNI.java`
- `src/org/sosy_lab/java_smt/solvers/leansmt/LeanSMTConstants.java`

## Notes

- Keep ownership-sensitive JNI behavior in `leansmt.i` typemaps so regeneration is reproducible.
- `leansmt_extra_jni.c` remains hand-maintained and is not generated by SWIG.
  It is intentionally small and only covers custom conversions for the extra `symbol`, `apply`,
  and `declare_fun` entry points.
- After regeneration, always inspect diffs before commit.
